# Plan 02.01 — Consolidate entity types into `src/lib/types/`

## Goal

Stand up `src/lib/types/` as the canonical type-import location for the rest of the app. Each entity gets a thin file that re-exports the type from its seed file (single source preserved).

## Why

DATA-01 #3: "Every domain entity is declared as an exported TypeScript type used by both reads and writes — no page redeclares its own shape."

## Actions

1. Create directory `src/lib/types/` and `src/lib/types/tariff/`.
2. For each of the **11 main entities** + **5 tariff entities**, create `<entity>.ts` containing `export type { ... } from '@/data/seed/<path>'` for every exported type/interface in that seed file.
3. Create `src/lib/types/index.ts` re-exporting every entity file.
4. Run `npx tsc --noEmit` — must remain at exit 0.
5. Commit: `feat(02-01): add src/lib/types/ canonical type barrel`.
