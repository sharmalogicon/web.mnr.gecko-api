# Plan 02.02 — Repository layer with port/adapter shape

## Goal

Stand up `src/lib/repos/` with one interface + InMemory implementation per entity, plus the single wiring file (`src/lib/repos/index.ts`) referenced by DATA-02. Also ship one HTTP-stub example to document the future swap path.

## Why

- DATA-01 #1: pages render from a single shared data source (repos hold the seed reference, all consumers reach the same array).
- DATA-02 #2: HTTP swap is a single-wiring-file change (the wiring file exists; the swap shape is documented + demonstrated).

## Per-entity repo template

```ts
// src/lib/repos/equipment.ts
import { equipment as seedEquipment } from '@/data/seed/equipment';
import type { EquipmentRecord } from '@/lib/types';

export interface EquipmentRepo {
  list(): EquipmentRecord[];
  get(id: string): EquipmentRecord | undefined;
}

export class InMemoryEquipmentRepo implements EquipmentRepo {
  list(): EquipmentRecord[] {
    return seedEquipment;
  }
  get(id: string): EquipmentRecord | undefined {
    return seedEquipment.find((e) => e.id === id);
  }
}
```
