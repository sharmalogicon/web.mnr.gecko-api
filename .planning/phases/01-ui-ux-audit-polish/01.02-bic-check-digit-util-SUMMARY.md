# Phase 1 Plan 02: BIC ISO 6346 Check-Digit Util Summary

Pure-function BIC ISO 6346 check-digit utility shipped via TDD (RED→fix-anchors→GREEN), exposing `computeCheckDigit` and `isValidContainerNumber` from `src/lib/iso6346/check-digit.ts`. The algorithm uses the BIC letter-value table (skipping multiples of 11), positional weights 2^0..2^9, and the `(sum % 11) % 10` formula — verified independently against the canonical BIC reference example `CSQU305438 → 3`.

## What Was Built

A single pure-function module (no I/O, no external deps) implementing the Bureau International des Containers ISO 6346 check-digit algorithm. Two exports:

- `computeCheckDigit(ownerCodeAndSerial)` — takes a normalised 10-char BIC prefix (4 letters + 6 digits) and returns the 0–9 check digit. Throws on malformed shape.
- `isValidContainerNumber(full11)` — takes an 11-char container number, returns `true` iff the check digit matches what `computeCheckDigit` would produce. Returns `false` on any malformed input (rather than throwing) so it is safe to use in form-validation contexts (Phase 3 EQUIP-04).

Both inputs are normalised (uppercased, whitespace-stripped) before shape validation. The module is import-safe at build time — plan 01.07 will call `isValidContainerNumber` over every seed container number before the seed file commits.
