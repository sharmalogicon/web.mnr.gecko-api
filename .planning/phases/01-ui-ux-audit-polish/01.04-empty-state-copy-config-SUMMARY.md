# Phase 1 Plan 04: Empty-State Copy Config Summary

Per-route Phase-1 user-facing copy ships as a single typed TS registry ‚Äî three const exports (`emptyStateCopy`, `loadingLabels`, `errorCopy`) plus three resolution helpers ‚Äî giving plans 08 and 09 zero copy decisions to make when wiring 47 routes.

### Type contract

```ts
type StateVariant = 'empty' | 'filter-empty' | 'not-found' | 'error';

interface CopyConfig {
  icon: string;
  title: string;
  description: string;        // contains {ID} placeholder for not-found
  primary?:   { label: string; href: string };
  secondary?: { label: string; href: string };
}

type RouteCopyMap = Partial<Record<StateVariant, CopyConfig>>;
```

Matches `EmptyStateProps` from `src/components/ui/EmptyState.tsx` (plan 01.01) so call-sites are pure spread / destructure ‚Äî no adapter layer.

### Key Conventions

1. **Route-path-as-key:** Keys are the Next.js route path, including dynamic segments (`/equipment/[id]`). Plans 08/09 look up by `usePathname()` or hardcoded route literal.

2. **`{ID}` placeholder:** Not-found descriptions and `?q={ID}` hrefs carry the literal string `{ID}`. Callers substitute at render time:
   ```ts
   description.replace('{ID}', params.id)
   ```
   Why: keeps the config a pure serialisable string (D-04) so the i18n pass can move it byte-for-byte to a locale file.

3. **`_fallback` for unmapped routes:** `errorCopy._fallback` is the universal error message. The map type is `Record<string, ErrorCopyEntry> & { _fallback: ErrorCopyEntry }` ‚Äî TS makes `errorCopy._fallback` non-optional, so `getErrorCopy()` is total. (W-6 fix from plan-checker.)

4. **Routes intentionally omitted from emptyStateCopy:**
   - All `/*/new` and `/*/[id]/edit` form pages ‚Äî forms have no list, so no empty state. They use `errorCopy._fallback` on submit failures.
   - `/settings/profile`, `/settings/notifications`, `/settings/language`, `/settings/company`, `/settings/display` ‚Äî single-form pages, not lists.
   - `/storage` has no filter-empty/empty in emptyStateCopy directly only `empty` ‚Äî storage view is a derived list of containers on hold; `filter-empty` is included as the typical pattern.
   - Public pages `/`, `/login`, `/forgot-password` ‚Äî handled by their own component-local copy, not registry-driven.

## Note for Phase D (i18n)

This file is the single i18n swap target for Phase-1 user-facing copy:

```ts
// Before (Phase 1):
title: 'No repair jobs yet',

// After (Phase D):
title: t('emptyStates./repair.empty.title'),
```

One file, mechanical sweep, zero risk of missing strings across 47 routes. That's the entire reason for the centralisation (D-04). The `{ID}` placeholder convention also maps cleanly to ICU MessageFormat: `t('emptyStates./repair/[id].not-found.description', { ID: params.id })`.
