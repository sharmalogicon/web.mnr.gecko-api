## Deviations from Plan

### Scope-boundary observations (not deviations)

**Pre-existing `<h1>` in non-PageHeader pages — left untouched per file-discipline:**

The plan's Task 2 acceptance criterion (lines 308-311) asks for `grep -rln "<h1" src/app/` to return only public-page hits. However, several other AppShell-wrapped pages contain custom `<h1>` elements that were NEVER PageHeader sites and therefore out of scope for this plan:

- `src/app/dashboard/page.tsx`
- `src/app/repair/page.tsx`, `src/app/repair/[id]/page.tsx`, `src/app/repair/new/page.tsx`
- `src/app/tariff/**` (12 files)

These pages were never in the PageHeader discovery grep (none imported PageHeader). The plan's `file-ownership note` (lines 41-46) is explicit that plans 06/08/09 own these files. Adding `<h1>` removal in those files now would violate the line-discipline rule and risk merge conflicts with the Wave 3 plans.

**For ALL 20 pages this plan modified, `<h1>` count is zero** — confirmed via per-file grep loop. The "no page-body H1" UI-SPEC sec14 quality gate passes for every page this plan touched.

CSS files (`globals.css`, `gecko_design_system_components.css`) also contain `<h1>` strings — these are CSS selectors, not React elements; not violations.

### Implementation choices

- Kept original action JSX shape (lucide icons via `<Plus className="mr-2 h-4 w-4" />`, `<Button>` shadcn components) rather than converting to `gecko-btn gecko-btn-primary gecko-btn-sm` + `<Icon name="..." />`. The plan's example AFTER block (line 271-278) shows the `gecko-btn` form as the target style, BUT:
  - The plan's file-discipline rule (lines 291-293) restricts edits to "PageHeader import line + the `<PageHeader ... />` JSX block + any orphan `<h1>` element"
  - Converting Button/Icon variants for 14 pages would touch many additional lines and risk colliding with plans 06/08/09 in Wave 3
  - The `mnr-page-actions` flex container handles layout; the inner button styling is orthogonal and can be migrated to `gecko-btn` in a follow-up plan once Wave 3 lands
- Title case preserved in button text ("New Invoice", "Add Part") rather than converted to sentence case ("New invoice", "Add part") for the same scope-boundary reason; UI-SPEC sec1 voice cleanup can run as a separate sweep
- Settings layout (`src/app/settings/layout.tsx`) PageHeader removed cleanly — the per-page `<h2>` titles inside each settings sub-route are unaffected
