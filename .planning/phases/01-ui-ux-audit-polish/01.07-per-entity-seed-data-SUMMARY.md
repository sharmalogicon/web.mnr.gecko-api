## Vitest CI guard (D-10 implementation)

`src/data/seed/equipment.validation.test.ts` runs on every `npm test`:

1. Every equipment record's `id` passes `isValidContainerNumber`. A regression (wrong check digit committed) fails the test by listing the bad IDs.
2. `equipment.length >= 18` — coverage floor enforcement.
3. Set of `category` values includes DRY, TANK, REEFER — the 3 primary domain categories.
4. All 7 D-11 anchor IDs present — anchor regression detection.

Plus the existing 13 BIC-util tests at `src/lib/iso6346/check-digit.test.ts` — total 17 tests in the suite.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added vitest.config.ts for `@/*` path alias**

- **Found during:** Task 3 (running `npm test` for the first time after creating `equipment.validation.test.ts`).
- **Issue:** `equipment.ts` imports `from '@/lib/iso6346/check-digit'` (plan-mandated `key_links` pattern). vitest cannot resolve `@/` without a `vitest.config.ts` declaring the alias. The test file failed to load, blocking the CI guard from running.
- **Fix:** Created `vitest.config.ts` with `resolve.alias` mapping `@` → `./src` (mirrors `tsconfig.json` paths). Did NOT regress `equipment.ts`'s `@/`-style import (which is the plan's `key_links` convention).
- **Files modified:** `vitest.config.ts` (new file).
- **Commit:** 388ffc3.

**2. [Rule 1 - Bug] Plan's repair/survey/cleaning examples used OLD/INVALID anchor IDs**

- **Found during:** Task 1 authoring.
- **Issue:** The plan body's repair.ts / survey.ts / cleaning.ts snippets referenced container IDs like `MSKU2345675` and `TCNU8453218` (the OLD check digits, before the 2026-05-18 D-11 correction). Those IDs do NOT BIC-validate. The objective explicitly required using the CORRECTED IDs (`MSKU2345671`, `TCNU8453210`, etc.).
- **Fix:** Authored all FK references using the corrected anchor IDs throughout repair.ts / survey.ts / cleaning.ts / storage.ts / modification.ts / emergency.ts. The plan's `equipment.validation.test.ts` snippet ALSO used the OLD anchors; the as-written test file uses the corrected anchors.
- **Files modified:** Every file authored in Tasks 1, 2, 2b, 3.
- **Commits:** fd62eec, 7a271db, 90e950a, 388ffc3.
