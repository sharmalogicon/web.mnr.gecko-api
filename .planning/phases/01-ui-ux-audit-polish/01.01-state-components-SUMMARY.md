# Phase 1 Plan 01.01: State Components Summary

Shipped the five shared state primitives that the entire phase depends on: `EmptyState` (4 variants + primary/secondary CTAs), `ErrorState` (composed atop EmptyState with `<details>` disclosure + placeholder correlation ID + Copy details button), `LoadingState` exports (`TableSkeleton`, `KpiTileSkeleton`, `DetailSpinner`), a tiny pure-function `levenshtein` / `nearestReference` helper, and the UI-SPEC §13 overlay CSS block (§6.4 — variant icon tints, compact card/empty paddings, page-actions row, error-disclosure styling, KPI compact). All 47 routes in Waves 3 & 4 now have the building blocks needed for the audit wiring; no downstream wiring is blocked.

## What Was Built

- **`src/components/ui/EmptyState.tsx`** (90 LOC) — Ported from TOS sibling repo `web.tos.gecko-api/src/components/ui/EmptyState.tsx`, then extended per UI-SPEC §5.1: added `EmptyStateVariant` union (`empty | filter-empty | not-found | error`), `primary?` / `secondary?` typed CTA props (`{ label, href?, onClick? }`), `compact` prop for in-card use, and per-variant default icon (`box` / `search` / `fileX` / `alertCircle`). CTAs render as `next/link` `<Link>` when `href` is set, `<button>` otherwise. Wrapper className suffix `gecko-empty-state-${variant}` drives the §6.4 CSS icon-disc tints. XSS-safe (no `dangerouslySetInnerHTML`).
- **`src/components/ui/ErrorState.tsx`** (75 LOC) — Composes `<EmptyState variant="error" ...>` and appends a native `<details className="gecko-error-disclosure">` block showing Code / Correlation / Message rows + Copy details button. Retry CTA wired via `onRetry?: () => void` prop (omitted from EmptyState's primary slot when undefined). Exports `placeholderCorrelationId()` helper generating `MNR-{8-hex-chars}` via `crypto.getRandomValues(new Uint8Array(4))`. Clipboard write uses `navigator.clipboard.writeText` with JSON.stringify (T-01-01 mitigation — payload is internal-only).
- **`src/components/ui/LoadingState.tsx`** (90 LOC) — Three named exports: `TableSkeleton({ columns, rows })` renders inside a `gecko-card gecko-card-compact` shell with a grid of `gecko-skeleton gecko-skeleton-text` bars; `KpiTileSkeleton()` matches the §6 KPI tile shape with label + icon disc + value + sparkline skeletons; `DetailSpinner({ label })` centres the existing `.gecko-spinner.gecko-spinner-lg` with a label below in `--gecko-text-secondary`. No new `@keyframes`, no new `animation:` rules — all animation derives from `.gecko-skeleton`'s built-in shimmer.
- **`src/lib/levenshtein.ts`** (62 LOC) — Pure utility, zero deps. `levenshtein(a, b)` classic two-row DP edit distance; `nearestReference(needle, candidates)` returns the closest candidate where Levenshtein ≤ 2 OR a fallback matching numeric prefix of ≥ 4 digits (per D-03). Returns `null` (typed Maybe) when no match.
- **`src/app/gecko_mnr_overlay.css`** (+67 LOC at EOF) — Appended UI-SPEC §13 verbatim as §6.4: four `.gecko-empty-state-{variant} .gecko-empty-state-icon` tint rules, `.gecko-empty-state-compact` padding, `.gecko-card-compact` header/body/footer/title rules, `.mnr-page-actions` + `.mnr-page-actions-spacer` flex row, `.gecko-error-disclosure` + `.gecko-error-disclosure-body` disclosure styling, and `.gecko-stat-card-compact` KPI tile compact override. TOC at top of file gained `6.4` entry. §6.1, §6.2, §6.3 byte-identical. All values resolve to existing `--gecko-*` tokens — zero new hex literals in the §6.4 block.

Downstream pages only need to set the `variant` prop on `<EmptyState>` — no per-page CSS work.

## D-05 Override Resolution (REQUIRED PER PLAN OUTPUT CONTRACT)

**CONTEXT.md D-05 said:** *"Skeleton screens use a subtle shimmer animation (CSS-only `@keyframes shimmer` driven by gray-100 → gray-200 background slide); ~1.5s ease-in-out."*

**UI-SPEC §8.2 supersedes:** **1.8s linear infinite** is the locked timing — already shipped in `gecko_design_system_components.css:553-568` via the `@keyframes gecko-shimmer` and `.gecko-skeleton { animation: gecko-shimmer 1.8s infinite linear; }` rule. The shared design system constraint with the TOS sibling repo wins over the discussion-phase suggestion. Mutating to 1.5s ease-in-out would diverge MNR from TOS and violate UI-SPEC's "no new design dependencies" rule.

**Implementation honours UI-SPEC.** Phase 1 does NOT redefine `@keyframes` anywhere; `LoadingState.tsx` composes the existing `.gecko-skeleton` class directly. Verified by `! grep @keyframes src/components/ui/LoadingState.tsx` (no matches) and `! grep animation: src/components/ui/LoadingState.tsx` (no matches).
