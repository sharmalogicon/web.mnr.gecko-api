<objective>
Eliminate `<PageHeader>` from all 20 import sites per UI-SPEC §1 / §7 (the "no big page-body H1" corrective). The AppShell header already shows the 15px page title (`src/components/layout/app-shell.tsx:333-349`). Pages that have action buttons get a new compact `<div className="mnr-page-actions">` inline row above content; pages without actions just start straight into content.

Purpose: UI-SPEC §1 fundamental rule #2 ("Page bodies must NOT render a big `<h1>` at the top"). This is the visual density unlock that brings MNR to TOS parity. Without this plan, even a perfectly-wired EmptyState ships next to a giant page-body `<h1>` and density fails the UI-SPEC §14 quality gate.

Output: 20 modified page files (PageHeader removed, action row added where appropriate), 1 deprecated component file kept on disk with a warning banner, 1 barrel-index edit.

Scope clarity:
- This plan handles PageHeader REMOVAL only.
- Plans 08 + 09 do the wider per-page wiring (state branches, seed imports). The 20 PageHeader sites overlap with the ~35 pages those plans touch, but PageHeader removal is mechanical and independent enough to ship in Wave 1 as a foundation (so plans 08+09 don't need to do this work too).
